## 1. Цель и задачи работы

**Цель.** Разработать программный модуль для создания и редактирования размерных линий в соответствии со стандартами ЕСКД, освоив принципы автоматического вычисления геометрических параметров и оформления чертёжной документации.

**Поставленные задачи.**

1. Проанализировать стандарт ГОСТ 2.307-2011 «Нанесение размеров и предельных отклонений».
2. Реализовать иерархию классов для трёх типов размеров: линейного, радиального, углового.
3. Разработать механизм ассоциативной связи размеров с геометрическими примитивами.
4. Обеспечить ручное редактирование размеров (грипперы, панель свойств).
5. Реализовать корректный экспорт в DXF без нарушения совместимости с другими CAD-программами.

## 2. Анализ стандарта ЕСКД ГОСТ 2.307-2011

ГОСТ 2.307-2011 регламентирует правила нанесения размеров на чертежах. Ключевые требования, заложенные в реализацию:

| Требование стандарта | Реализация |
| --- | --- |
| Размерные линии должны иметь стрелки на концах | `_draw_dim_arrow` — закрашенный треугольник через `arrow_polygon` (соотношение длины к ширине ~3:1) |
| Выносные линии выходят за размерную на 1…5 мм | Параметр `ext_overshoot` (по умолчанию 0.15 мировых единиц) |
| Между геометрией и началом выносной линии — небольшой зазор | Параметр `ext_offset` (по умолчанию 0.05) |
| Числовое значение размещается ПАРАЛЛЕЛЬНО размерной линии (п. 4.6) | `text_angle = atan2(dy, dx)` с нормализацией в `(-90°, 90°]` чтобы текст не оказался перевёрнут |
| Текст диаметра предваряется знаком ⌀ | `RadialDimension.label()` префиксует `'⌀'` для `kind='diameter'` |
| Текст радиуса предваряется буквой R | `RadialDimension.label()` префиксует `'R'` для `kind='radius'` |
| Угловые размеры — со значением в градусах с символом ° | `AngularDimension.label()` добавляет `'°'` |
| Размер диаметра при выносе из окружности рекомендуется на полке | Реализована опция `with_shelf` (по умолчанию включена для диаметра) |
| Стрелки касательны к размерной дуге (для угловых) | В `_draw_dim_angular` стрелки направлены вдоль касательной к дуге |
| Текст не должен пересекаться размерной линией | Текст автоматически смещается от размерной линии на `text_height * 0.7` |

## 3. Архитектура решения

Решение разделено на два модуля:

- **`dimensions.py`** — независимая библиотека, содержащая иерархию классов размеров. Не имеет зависимостей от Tkinter или приложения.
- **`main4.py`** — основное Tkinter-приложение. Подключает классы размеров, реализует UI, обработку кликов, отрисовку, грипперы, привязки.

Размеры хранятся **в одном общем массиве** `self.primitives` наряду с прочими примитивами (отрезок, окружность, дуга, ломаная и т.д.). Это упрощает:

- единый цикл отрисовки (`redraw_all`);
- единый список в боковой панели;
- единые операции выбора/удаления/дублирования;
- единый формат экспорта/импорта.

Размер хранится в виде словаря:

```python
{
    'type':       'dim_linear' | 'dim_radial' | 'dim_angular',
    'params':     { ... }      # сериализованные параметры размера
    'color':      '#FFFFFF',
    'style_name': 'Сплошная тонкая',
    'name':       'Размер лин. 3',
    'id':         42,
}
```

Объект-размер реконструируется из `params` фабрикой `make_dimension(type, params)` каждый раз, когда нужны методы `measure()`, `label()`, `geometry()`. Это позволяет хранить состояние в простом dict-формате (легко сериализуется в JSON для DXF-сайдкара) и при этом пользоваться объектно-ориентированной логикой.

## 4. Иерархия классов размеров

Файл [`dimensions.py`](dimensions.py).

```
Dimension (абстрактный базовый класс)
    ├── LinearDimension     KIND = 'dim_linear'
    ├── RadialDimension     KIND = 'dim_radial'   (kind = 'radius' | 'diameter')
    └── AngularDimension    KIND = 'dim_angular'
```

### 4.1. Базовый класс `Dimension`

Содержит:

- словарь `DEFAULTS` со всеми параметрами оформления и их значениями по умолчанию;
- общий конструктор, читающий `**style` и устанавливающий атрибуты по `DEFAULTS`;
- атрибут `assoc` — список ассоциативных ссылок на примитивы-источники;
- абстрактные методы `measure()` и `geometry()`;
- метод `label()`, формирующий числовую подпись (или `text_override`);
- метод `to_params()` для сериализации в dict.

### 4.2. `LinearDimension`

Линейный размер с тремя ориентациями: `horizontal`, `vertical`, `aligned`.

**Поля:**

| Поле | Назначение |
| --- | --- |
| `p1`, `p2` | Точки измерения (вершины измеряемого объекта) |
| `dim_pos` | Любая точка, через которую проходит размерная линия |
| `orientation` | Направление размерной линии: `horizontal` / `vertical` / `aligned` |

**Ключевые методы:**

- `direction()` → единичный вектор размерной линии (для `horizontal` всегда `(1,0)`; для `vertical` — `(0,1)`; для `aligned` — нормированный `p2-p1`);
- `normal()` → перпендикуляр (для отступа текста);
- `project(pt)` → проекция точки на размерную линию (вычисление точек, где выносные линии пересекают размерную);
- `measure()` → длина проекции `p2-p1` на направление размера;
- `geometry()` → словарь с расчётной геометрией для отрисовки (`e1`, `e2`, `direction`, `normal`).

### 4.3. `RadialDimension`

Радиальный размер: радиус (R) или диаметр (⌀).

**Поля:**

| Поле | Назначение |
| --- | --- |
| `cx`, `cy`, `r` | Параметры окружности/дуги |
| `angle_deg` | Угол радиальной линии (от центра) |
| `kind` | `'radius'` либо `'diameter'` |
| `text_pos` | Положение надписи (необязательное, может быть вне окружности) |
| `with_shelf` | Включить полку-выноску (по умолчанию `True` для диаметра) |

**Особенности:**

- `measure()` для диаметра возвращает `2·r`;
- `label()` префиксует `'⌀'` или `'R'`;
- `geometry()` возвращает точку на окружности (`on_circle`), противоположную точку (`opposite`), позицию текста.

### 4.4. `AngularDimension`

Угловой размер между двумя лучами от вершины.

**Поля:**

| Поле | Назначение |
| --- | --- |
| `vertex` | Вершина угла |
| `p1`, `p2` | Точки на 1-м и 2-м лучах |
| `arc_radius` | Радиус размерной дуги |

**Особенности:**

- `angles()` → углы лучей в радианах (через `atan2`);
- `measure()` → меньший угол между лучами в градусах;
- `geometry()` → дополнительно вычисляет средний угол `mid` (для размещения текста) и знак направления дуги `sign`.

### 4.5. Фабрика и проверка типа

```python
def make_dimension(prim_type, params):
    if prim_type == LinearDimension.KIND:  return LinearDimension.from_params(params)
    if prim_type == RadialDimension.KIND:  return RadialDimension.from_params(params)
    if prim_type == AngularDimension.KIND: return AngularDimension.from_params(params)
    raise ValueError(...)

def is_dimension_type(prim_type):
    return prim_type in (LinearDimension.KIND, RadialDimension.KIND, AngularDimension.KIND)
```

## 5. Параметры оформления

Все параметры заданы в `Dimension.DEFAULTS`:

| Параметр | Назначение | Значение по умолчанию |
| --- | --- | --- |
| `text_override` | Ручное переопределение надписи (если не None — игнорируется автоматически вычисленное значение) | `None` |
| `arrow_size` | Длина стрелки в мировых единицах | `0.30` |
| `text_height` | Высота шрифта подписи | `0.35` |
| `ext_overshoot` | Выход выносной линии за размерную | `0.15` |
| `ext_offset` | Зазор между геометрией и началом выносной | `0.05` |
| `dim_extension` | Выход размерной линии за выносные | `0.00` |
| `precision` | Знаков после запятой в подписи | `2` |
| `arrow_filled` | Закрашенные стрелки (False = контурные) | `True` |
| `text_color` | Цвет текста | `'#FFFFFF'` |
| `line_color` | Цвет линий размера | `'#FFFFFF'` |
| `ext_style` | Стиль выносных линий (имя из `line_styles`) | `'Сплошная тонкая'` |
| `dim_style` | Стиль размерной линии | `'Сплошная тонкая'` |
| `with_shelf` | Полка под надписью (для радиальных) | `False` |

В `SegmentApp.__init__` те же значения собраны в `self.dim_defaults` и применяются при создании новых размеров через UI. Пользователь может изменить эти значения через панель свойств выделенного размера.

## 6. Создание размеров (UI и логика кликов)

### 6.1. Кнопки инструментов

В левой панели в секции «ПРИМИТИВЫ САПР» добавлен подраздел **«РАЗМЕРЫ (ЕСКД 2.307)»** с четырьмя кнопками:

- 📏 **Линейный** (`dim_linear`)
- ⌖ **Радиус** (`dim_radial_r`)
- ⌀ **Диаметр** (`dim_radial_d`)
- ∠ **Угловой** (`dim_angular`)

При выборе типа справа активируется комбобокс «Метод» с вариантами подметодов (например, для линейного: «Выровненный» / «Горизонтальный» / «Вертикальный»).

### 6.2. Сценарии создания

#### Линейный размер (3 клика)

1. Указать первую точку измерения (можно с привязкой к концу/середине отрезка).
2. Указать вторую точку измерения.
3. Указать положение размерной линии (любая точка на ней).

Для метода «Горизонтальный»/«Вертикальный» проекция выполняется на горизонтальную/вертикальную ось через 3-ю точку. Для «Выровненного» — на линию, параллельную `p1-p2`.

#### Радиальный (2 клика)

1. Кликнуть **по самой окружности или дуге** — будет выбран ближайший подходящий примитив (`_pick_circle_or_arc`).
2. Указать положение текста — тем самым задаётся угол радиальной линии и (при необходимости) длина выноски.

#### Угловой (4 клика)

1. Указать вершину угла.
2. Указать точку на первом луче.
3. Указать точку на втором луче.
4. Указать положение размерной дуги (расстояние от вершины задаёт `arc_radius`).

### 6.3. Превью

В `update_prim_preview` для каждого типа размера показывается пунктирное превью текущего состояния создания, включая трассировку лучей и проекцию на размерную линию.

### 6.4. Подсказки

В `update_prim_hint` каждый шаг инструмента сопровождается текстовой подсказкой в нижней части панели («Укажите 1-ю точку измерения», «Укажите положение размерной линии» и т.д.).

## 7. Отрисовка размеров на холсте

Метод `_draw_dimension(prim, selected)` диспетчеризует на три специализированных рендерера:

### 7.1. `_draw_dim_linear`

Этапы отрисовки:

1. **Геометрия.** Получить `e1, e2` (точки на размерной линии — проекции `p1, p2`), направление и нормаль.
2. **Выносные линии** от каждой точки измерения до её проекции на размерную линию, с зазором `ext_offset` в начале и выходом `ext_overshoot` в конце.
3. **Размерная линия** от `e1` до `e2` (с расширением `dim_extension`).
4. **Стрелки** в `e1` и `e2`, направленные наружу.
5. **Текст** в середине размерной линии, смещён по нормали в сторону, противоположную геометрии. Поворот текста — вдоль направления размерной линии (нормализован в `(-90°, 90°]`).

### 7.2. `_draw_dim_radial`

- **Радиус**: линия от центра к точке `on_circle` + одна стрелка.
- **Диаметр**: линия от `opposite` до `on_circle` через центр + две стрелки.
- **Линия-выноска**: от `on_circle` до `text_pos`, если текст вынесен за окружность.
- **Полка**: горизонтальный отрезок длиной примерно `len(label) · text_height · 0.7`, начинается от `text_pos`, текст центрируется над ним.
- **Текст**: формата `R12.50` или `⌀25.00`.

### 7.3. `_draw_dim_angular`

- **Выносные лучи** — продолжения `vertex → p1` и `vertex → p2` до радиуса `R + ext_overshoot`.
- **Размерная дуга** — интерполяция между углами `a1` и `a2` (выбирается меньшая дуга, если угол развёрнут более 180°).
- **Стрелки** на концах дуги, направленные касательно (`tan = (-sin(a)·s, cos(a)·s)`).
- **Текст** смещён по нормали от середины дуги, повёрнут по касательной к дуге в этой точке.

### 7.4. Контрольные точки (грипперы)

При `selected=True` каждый рендерер дополнительно рисует синие квадраты в позициях, возвращаемых `_get_grippers(prim)` — это места, за которые можно «схватить» размер мышью.

## 8. Привязки (snap)

В системе доступны 6 типов привязок (галочки в секции «ПРИВЯЗКИ»):

| Привязка | Что снапит | Реализация |
| --- | --- | --- |
| **Конец** | Концы отрезков, ломаных, дуг, углы прямоугольников и многоугольников, квадранты окружностей | `_collect_snap_candidates` |
| **Середина** | Середины сегментов | `_collect_snap_candidates` |
| **Центр** | Центры окружностей, дуг + начало координат | `_collect_snap_candidates` |
| **Пересечение** | Точки пересечения любых двух «линейных» или «круговых» объектов | `_collect_intersection_candidates` |
| **Перпендикуляр** | Нога перпендикуляра, опущенного из «опорной» точки на сегмент | `_collect_perp_candidates` |
| **Касательная** | Касательная к окружности/дуге, проведённая из «опорной» точки | `_collect_tangent_candidates` |

### 8.1. Поиск ближайшей точки привязки

В `find_snap_point(sx, sy, world_x, world_y)`:

1. Радиус захвата = 20 пикселей в мировых единицах (учёт текущего масштаба).
2. Перебираются все источники-кандидатов; ближайший к курсору в пределах радиуса побеждает.
3. Для каждой найденной точки сохраняется не только координата, но и:
   - `snap_label` — название («Конец», «Касательная» и т.д.) для маркера;
   - `snap_ref` — `{'prim_id', 'src_key'}` для последующей ассоциативной связи.

### 8.2. Опорная точка для касательной/перпендикуляра

Эти привязки требуют второй точки — «откуда» вести линию. Выбирается:

- при создании примитива — последняя накопленная точка (`prim_points[-1]`);
- при перетаскивании гриппера — *противоположный* конец того же примитива (метод `_drag_from_point`).

### 8.3. Подавление self-snap

Когда пользователь тащит гриппер, перетаскиваемый примитив исключается из всех источников снапов:

```python
skip_idx = self.drag_ctrl_pt['idx'] if self.drag_ctrl_pt else None
for i, prim in enumerate(self.primitives):
    if i == skip_idx:
        continue
    ...
```

Иначе курсор «прилипал» бы к точке, которую сам же двигает.

### 8.4. Пересечения

Алгоритм `_collect_intersection_candidates(wx, wy, snap_r)`:

1. Извлечь из всех примитивов:
   - **отрезки** (включая стороны прямоугольников и многоугольников, рёбра ломаных);
   - **окружности и дуги** (с угловым диапазоном для дуг).
2. Перебрать все пары: отрезок-отрезок, отрезок-окружность, окружность-окружность.
3. Для каждой пары вычислить точки пересечения (см. ниже) и оставить только те, что попали в `snap_r` от курсора.

| Пара | Метод |
| --- | --- |
| отрезок-отрезок | `_seg_seg_intersect` через детерминант |
| отрезок-окружность | `_seg_circle_intersect` через квадратное уравнение |
| окружность-окружность | `_circle_circle_intersect` через геометрию двух кругов |

Для дуг проверяется попадание точки в угловой диапазон через `_angle_in_arc`.

### 8.5. Касательная

Метод `_collect_tangent_candidates(from_pt, wx, wy, snap_r)`:

```python
base_angle = atan2(fy - cy, fx - cx)
half = acos(r / dist)             # угол между from_pt-центр и радиусом-к-точке-касания
for sign in (+1, -1):
    touch_angle = base_angle + sign * half
    touch = (cx + r·cos, cy + r·sin)
    # тангенциальная прямая = from_pt → touch
    # снап-кандидат = проекция курсора на эту прямую
```

Это позволяет «вытягивать» отрезок дальше точки касания — снап скользит вдоль касательной прямой, как в AutoCAD/КОМПАС.

### 8.6. Перпендикуляр

`_collect_perp_candidates`: для каждого сегмента вычисляется нога перпендикуляра, опущенного из `from_pt`. Результат — точка на сегменте, гарантирующая угол 90° между новым отрезком и существующим.

## 9. Ассоциативная связь

### 9.1. Идея

Каждому примитиву при создании присваивается уникальный `id` (поле `_next_prim_id` инкрементируется). Размеры могут хранить ссылки `{'point_key': '...', 'prim_id': N, 'src_key': '...'}` в массиве `assoc`. При каждой перерисовке размер «подтягивает» актуальные координаты из примитива-источника.

### 9.2. Захват ссылок при создании

Когда пользователь кликает с активной привязкой, и снап выдаёт `snap_ref`, эта ссылка сохраняется в `self.creation_mode['snap_refs']`. После завершения создания размера соответствующие ссылки записываются в `assoc`:

```python
# Линейный размер: первая точка цепляется за источник snap'а
if refs[0]: assoc.append({'point_key': 'p1', **refs[0]})
if refs[1]: assoc.append({'point_key': 'p2', **refs[1]})
```

### 9.3. Резолвинг

`_resolve_assoc_point(ref)` находит примитив по `id` и возвращает актуальные координаты по ключу `src_key`:

| Тип источника | Ключи |
| --- | --- |
| `segment_mouse` | `p1`, `p2`, `mid` |
| `polyline` | `pt0`, `pt1`, ... |
| `circle`, `arc` | `center`, `circle` |
| `rect` | `corner0`...`corner3` |
| `polygon` | `vert0`, `vert1`, ... |

### 9.4. Автоматическое обновление

`_refresh_dim_assoc(prim)` вызывается перед отрисовкой каждого размера:

```python
for ref in params.get('assoc', []):
    pt = self._resolve_assoc_point(ref)
    if pt is None:
        continue
    key = ref['point_key']
    if   key in ('p1', 'p2', 'vertex'):
        params[key] = pt
    elif key == 'circle':
        params['cx'], params['cy'] = pt
        # для радиального — также подтянуть r
```

В результате при перемещении/изменении исходного примитива размер обновляется автоматически.

### 9.5. Разрыв связи при ручном редактировании

Если пользователь перетаскивает гриппер размера, для соответствующей точки ассоциативная ссылка удаляется (он явно сказал «здесь»):

```python
elif t == 'dim_linear':
    if key in ('p1', 'p2', 'dim_pos'):
        p[key] = tuple(new_pt)
        p['assoc'] = [a for a in p.get('assoc', []) if a.get('point_key') != key]
```

## 10. Грипперное редактирование

### 10.1. Концепция

«Гриппер» — небольшой синий квадрат в характерной точке выделенного примитива, за который можно «схватиться» мышью и переместить, изменив параметр.

### 10.2. Состав грипперов по типам

| Тип примитива | Грипперы |
| --- | --- |
| `segment_mouse` | `p1`, `p2` |
| `polyline` | `pt0`, `pt1`, ... `ptN` |
| `circle` | `center`, `edge` |
| `arc` | `center`, `start`, `end` |
| `rect` | `corner0`...`corner3` |
| `ellipse` | `center`, `rx`, `ry` |
| `polygon` | `center`, `vertex` |
| `spline` | `pt0`, `pt1`, ... |
| `dim_linear` | `p1`, `p2`, `dim_pos` |
| `dim_radial` | `center`, `on_circle`, `text_pos` |
| `dim_angular` | `vertex`, `p1`, `p2`, `arc_mid` |

### 10.3. Жизненный цикл перетаскивания

```
on_left_click:
    if grip := _hit_grip(event.x, event.y):
        self.drag_ctrl_pt = grip
on_left_drag:
    if self.drag_ctrl_pt:
        wx, wy = screen_to_world(event.x, event.y)
        if snap → wx, wy = snap_point
        _move_gripper(prim, key, (wx, wy))
        redraw_all()
on_left_release:
    if self.drag_ctrl_pt:
        # Финализация
        build_prim_props_panel(prim)
        update_prims_list()
        self.drag_ctrl_pt = None
```

### 10.4. Реакция на перетаскивание

Метод `_move_gripper(prim, key, new_pt)` интерпретирует движение в зависимости от типа и ключа:

- Двигаем `edge` окружности → меняется `r`.
- Двигаем `start` дуги → меняется и `start_angle`, и `r`.
- Двигаем `corner0` прямоугольника → пересчитываются все 4 угла, противоположный остаётся фиксированным.
- Двигаем `vertex` многоугольника → пересчитываются все вершины (с поворотом).
- Двигаем `arc_mid` углового размера → меняется `arc_radius`.
- и т.д.

### 10.5. Привязки при перетаскивании

Все привязки доступны и при перетаскивании грипперов. См. раздел [8.2](#82-опорная-точка-для-касательнойперпендикуляра).

## 12. Полка для диаметра

По ЕСКД при выносе значения диаметра за окружность размер размещается на полке-выноске — горизонтальной линии под надписью.

**Реализация:**

- В `Dimension.DEFAULTS` добавлен флаг `with_shelf` (по умолчанию `False`).
- При создании размера типа «Диаметр» автоматически выставляется `with_shelf=True`.
- В `_draw_dim_radial` при `with_shelf and text_outside`:
  ```python
  shelf_len = max(len(label) * text_height * 0.7, text_height * 2.0)
  sign = +1 if text_pos.x ≥ cx else -1
  shelf_end = (text_pos.x + sign * shelf_len, text_pos.y)
  draw_line(text_pos, shelf_end)
  draw_text((text_pos + shelf_end) / 2 + above)
  ```
- В панели свойств радиального размера — чекбокс **«Полка-выноска»**.

## 13. Экспорт и импорт DXF

### 13.1. Стратегия

Прямая запись DIMENSION-сущностей в DXF R12 требует громоздкой настройки DIMSTYLE/BLOCKS, и не все CAD-программы корректно их рендерят. Принят компромисс **«взорванная геометрия + сайдкар»**:

1. **В сам `.dxf`** размеры записываются как стандартные сущности (LINE, ARC, SOLID, TEXT) на отдельном слое **«Размеры»**. Это гарантирует корректное отображение в любой CAD-программе (AutoCAD, КОМПАС, nanoCAD, LibreCAD, QCAD, FreeCAD, BricsCAD).
2. **Рядом с DXF** создаётся файл `<имя>.dim.json` с исходными параметрами размеров. Другие CAD-программы об этом файле не знают — он не нарушает их работу.

### 13.2. Развёртка размера в DXF

| Часть размера | Сущность DXF |
| --- | --- |
| Выносные линии, размерная линия, выноска, полка | `LINE` |
| Размерная дуга углового размера | `ARC` |
| Стрелки | `SOLID` (закрашенный треугольник) |
| Числовая надпись | `TEXT` (стиль `STANDARD`, добавлена таблица `STYLE`) |

В `_emit_dim_linear`/`_emit_dim_radial`/`_emit_dim_angular` зеркалится логика отрисовки, но вместо вызова `canvas.create_*` пишутся теги DXF.

### 13.3. JSON-сайдкар

Структура файла `<имя>.dim.json`:

```json
{
  "format": "geomod-dimensions",
  "version": 1,
  "dimensions": [
    {
      "type": "dim_linear",
      "name": "Размер лин. 3",
      "color": "#FFFFFF",
      "style_name": "Сплошная тонкая",
      "id": 7,
      "params": { "p1": [0, 0], "p2": [10, 0], "dim_pos": [0, 3], ... }
    },
    ...
  ]
}
```

### 13.4. Поведение при импорте

```python
sidecar_dims = _read_dim_sidecar(filepath)
skip_dim_layer = bool(sidecar_dims)

for entity in dxf:
    if skip_dim_layer and entity.layer == 'Размеры':
        continue   # геометрию слоя пропускаем — будет восстановлена из сайдкара
    primitive = convert(entity)
    primitives.append(primitive)

for d in sidecar_dims:
    primitives.append(rebuild_dimension(d))
```

Сценарии:

- **Файл создан нашим приложением, открывается им же**: размеры восстанавливаются из сайдкара со всеми ассоциативными связями, как редактируемые объекты.
- **Файл редактировали в стороннем CAD**: сайдкар отсутствует или устарел → геометрия слоя «Размеры» приходит как обычные примитивы (`segment_mouse`, `arc`, ...). Визуальная картинка сохранена, но размеры теряют редактируемость.

### 13.5. Валидация

Файл прошёл проверку парсером `ezdxf 1.4.3`:
- Версия DXF: AC1009 (R12)
- Слои: `0`, `Размеры`, `Сплошная_основная`, `Сплошная_тонкая`, `Defpoints`
- Сущности в Modelspace: LINE, CIRCLE, SOLID, TEXT, ARC
- **0 ошибок, 0 исправлений**

## 15. Тестирование

### 15.1. Юнит-тесты классов

```python
ld = LinearDimension((0,0), (10,0), (0,2), orientation='horizontal')
assert ld.measure() == 10.0

rd = RadialDimension(0, 0, 5, 30, kind='diameter')

ad = AngularDimension((0,0), (10,0), (0,10), arc_radius=3)

```

### 15.2. Интеграционные сценарии

1. **Создание размеров через UI** — все три типа создаются без исключений, корректно отрисовываются.
2. **Ассоциативность**:
   - Отрезок (1,1)→(5,1) + линейный размер с `assoc[p1, p2]` → значение 4.0.
   - Изменяем `p2` → (10,1) → размер автоматически 9.0 ✓
   - Окружность r=2 + размер диаметра с `assoc[circle]` → значение 4.0.
   - Изменяем `r` → 4 → размер 8.0 ✓
3. **Грипперы**:
   - Все 4 гриппера углового размера (`vertex`, `p1`, `p2`, `arc_mid`) корректно перетаскиваются.
   - Радиальный — `text_pos` перетаскивает текст, обновляет угол.
   - Линейный — `dim_pos` перемещает размерную линию.
4. **Удаление**: Delete на выделенном объекте удаляет его, не срабатывает в текстовых полях.
5. **DXF round-trip**:
   - 3 размера → экспорт → импорт → 3 редактируемых размера.
   - Без сайдкара (имитация открытия в стороннем CAD) → 9 примитивов (геометрия размеров).
   - `ezdxf` валидирует файл с 0 ошибок.
6. **Привязки**:
   - Пересечение двух пересекающихся отрезков — точка ровно на пересечении.
   - Касательная из внешней точки скользит вдоль касательной прямой.
   - Перпендикуляр от точки на отрезок — точно на отрезке.
   - Self-snap при перетаскивании гриппера подавлен.
